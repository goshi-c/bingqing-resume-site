## 产品目标

把一页中文简历转成可访问、可阅读、可交互的个人作品网站，面向 AI 产品经理、数据产品、AI 应用产品岗位展示候选人的能力、经历、项目与自我评价。

## 用户与场景

- 面试官：快速判断候选人的岗位匹配度、项目真实性与联系方式。
- 候选人本人：投递时作为作品集入口，集中展示简历、Demo、GitHub 和数字分身。

## MVP 功能范围

1. 顶部导航与锚链接
   - 包含首页、优势、经历、项目、数字分身、联系入口。
   - 点击后平滑滚动到对应板块。

2. 首屏与视觉
   - 米色与橄榄色为主基调，整体简约、克制、有品质感。
   - 头像居中展示，使用彩色照片。
   - 首屏提供跳转到数字分身的按钮。

3. 简历内容展示
   - 展示基本信息、教育背景、实习经历、项目经历、技能与联系方式。
   - 在基本信息和教育信息之间穿插岗位理解与自我评价。
   - 文字基于简历真实内容，不新增未经证明的经历或指标。

4. 数字分身对话
   - 独立页面板块，不使用弹窗。
   - 支持围绕简历内容提问。
   - 公开页面不展示接口、模型和 API Key 配置。
   - 部署后通过后端函数调用 DeepSeek 模型。
   - 本地静态预览时提供本地预览回答，明确不冒充真实模型调用。

5. 响应式与动效
   - 适配手机与电脑。
   - 包含轻量滚动入场、按钮 hover、导航高亮、对话交互反馈。

## Not Now

- 不做账号系统。
- 不把 API Key 写入前端代码或仓库。
- 不做简历 PDF 导出。
- 不接入真实数据库或用户埋点。

## 验收标准

- 直接打开 `index.html` 能看到完整页面。
- 顶部导航锚链接可用，底部信息完整。
- 数字分身板块不是弹窗，能够本地预览回答；部署到 Vercel 并配置环境变量后具备调用 DeepSeek 能力。
- 页面在桌面与手机宽度下不出现明显文字溢出或布局重叠。
- 页面内容能体现 AI 产品经理方向、教育、实习、项目、技能和联系方式。
